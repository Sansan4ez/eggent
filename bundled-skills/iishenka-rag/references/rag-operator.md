## Что это

Оператор RAG-пайплайна. Держит векторную базу в актуальном состоянии по содержимому этой папки. Ты запускаешь его сам, когда добавил, изменил или удалил файлы.

Источник истины — `index.csv`. В нём по каждому файлу записаны путь, время модификации, статус и число чанков. Оператор сравнивает папку с `index.csv` и досинхронизирует только разницу.

## Как запустить

Из корня папки RAG:

```bash
python scripts/rag_sync.py            # применить изменения
python scripts/rag_sync.py --dry-run  # показать план, ничего не трогая
```

Или просто скажи в чате: **«прогони rag-оператор»** — и Claude выполнит шаги ниже.

Первый запуск: сначала поставь зависимости (один раз):

```bash
pip install -r scripts/requirements.txt
```

## Что делает каждый прогон

1. **Схема.** Гарантирует, что в БД есть расширение `pgvector`, таблица `{{RAG_TABLE}}`, индексы и функция `match_documents`. Идемпотентно: если всё уже есть, ничего не меняет.
2. **index.csv.** Читает его. Если файла нет, считает, что база пустая, и создаёт `index.csv` заново.
3. **Скан папки.** Рекурсивно проходит папку RAG, собирает файлы нужных форматов (`{{FILE_EXTENSIONS}}`) и их время модификации. Пропускает `scripts/`, `.env`, `index.csv`, `CLAUDE.md`, сам оператор и всё из `.ragignore`.
4. **Диф против index.csv:**
   - файл на диске, но не в `index.csv` → **NEW** → чанкинг + эмбеддинг + вставка.
   - время модификации на диске новее, чем в `index.csv` → **MODIFIED** → удалить старые строки по этому файлу из БД, затем векторизовать заново.
   - файл есть в `index.csv`, но пропал с диска → **DELETED** → удалить все строки по этому файлу из БД.
   - время не изменилось → пропустить.
5. **Запись index.csv.** Перезаписывает его с актуальными путями, временем, статусом `vectorized` и числом чанков.

Так база всегда отражает содержимое папки: добавил файл — он появится, изменил — переиндексируется, удалил — исчезнет.

## Как класть файлы

Клади документы прямо в эту папку (можно во вложенные подпапки, скан рекурсивный). Поддерживаются форматы: `{{FILE_EXTENSIONS}}`. Чтобы что-то исключить из индексации, добавь glob-шаблон в `.ragignore`.

## Поиск по базе

```bash
python scripts/rag_query.py "твой вопрос"
python scripts/rag_query.py "твой вопрос" -k 8
python scripts/rag_query.py "вопрос" --filter '{"file": "notes.md"}'
```

Из другого приложения поиск доступен как Supabase RPC `match_documents(query_embedding, match_count, filter)`.

## Настройки

Всё живёт в `.env` в корне этой папки: строка подключения, модель эмбеддингов и размерность, имя таблицы, размер чанка и overlap, список форматов, поля метаданных. Поменял модель эмбеддингов → поменяй `EMBEDDING_MODEL` и `EMBEDDING_DIM` вместе. Если сменил размерность на уже наполненной базе — очисти таблицу и прогони синк заново (старые векторы другой размерности не совместимы).

## Если что-то падает

- `psycopg2 not installed` / `openai package not installed` → `pip install -r scripts/requirements.txt`.
- `Missing required config DATABASE_URL` → заполни `.env`.
- PDF/DOCX пропускаются → поставь `pypdf` / `python-docx`.
- Ошибка про размерность вектора → `EMBEDDING_DIM` в `.env` не совпадает с моделью; выровняй и переиндексируй.
- DDL/`CREATE EXTENSION` падает на Supabase → используй прямую (session) строку подключения, а не transaction pooler.

## Правила

- `index.csv` — источник истины. Не редактируй его руками; оператор ведёт его сам.
- Один прогон = одна досинхронизация разницы. Полная переиндексация не нужна на каждый чих.
- `.env` не коммить (в нём ключи и пароль от БД).
